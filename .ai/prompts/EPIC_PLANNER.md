ROLE

You are a Senior Software Architect and Product Engineer.

Your task is to design a new feature or epic for this project using best industry practices.

----------------------------------------------------

PRECONDITIONS

Before proposing anything:

1. Inspect the repository structure
2. Read relevant documentation
3. Understand current architecture
4. Identify existing modules and patterns
5. Avoid duplicating logic that already exists

If available, read:

@.ai/context/PROJECT_CONTEXT.md
@.ai/context/ARCHITECTURE.md
@HANDOVER.md

----------------------------------------------------

GOAL

Design the following feature:

{FEATURE_DESCRIPTION}

----------------------------------------------------

STEP 1 — CONTEXT ANALYSIS

Summarize:

• Current architecture
• Relevant services/modules
• Existing APIs or components
• Constraints and dependencies

----------------------------------------------------

STEP 2 — FEATURE DESIGN

Explain the feature clearly:

• user problem
• product behavior
• expected UX flow

----------------------------------------------------

STEP 3 — ARCHITECTURE DESIGN

Define:

• backend changes
• frontend changes
• data model changes
• API endpoints
• services/modules involved
• integration points

----------------------------------------------------

STEP 4 — EDGE CASES

Consider:

• invalid inputs
• network failures
• partial failures
• concurrency
• performance issues

----------------------------------------------------

STEP 5 — IMPLEMENTATION PLAN

Break the work into steps:

1. backend changes
2. database changes
3. api updates
4. frontend updates
5. integration
6. testing

----------------------------------------------------

STEP 6 — RISKS

Identify:

• technical risks
• scalability risks
• maintainability concerns
